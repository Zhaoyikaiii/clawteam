# ClawTeam UI 设计规范 v0.1

> **飞书式结构 + AI 原生扩展 + 可工程落地 Design Token**

**更新时间**: 2026-02-14

---

## 一、设计原则 (Design Principles)

| 原则 | 说明 |
|------|------|
| **对话优先** | 聊天是主舞台，AI 不应喧宾夺主 |
| **结构化表达** | AI 输出必须"可扫描、可执行、可引用" |
| **可解释可信** | 所有 AI 行为都能追溯来源 |
| **企业级克制** | 风格稳定、冷静、不过度科技感 |

---

## 二、设计 Token (Design Tokens v0.1)

### 1. 色彩系统 (Color System)

#### 主色 (Primary)
用于：按钮 / @高亮 / 选中态

```
Primary-100  #E8F0FF
Primary-500  #3370FF
Primary-600  #245BDB
```

#### 中性色 (Neutral)
```
Background-Base     #FFFFFF
Background-Subtle  #F5F6F7
Background-Panel   #FAFAFB

Border-Light  #E5E7EB
Border-Default #D0D3D9

Text-Primary   #1F2937
Text-Secondary #4E5969
Text-Tertiary  #86909C
Text-Disabled  #C9CDD4
```

#### 状态色 (Semantic)
```
Success-500  #22B471
Warning-500  #FF9F1A
Danger-500   #F53F3F
Info-500     #3370FF
```

用于：Task 状态 / Approval 风险提示 / Agent 行为提示

### 2. 排版系统 (Typography)

#### 字体
系统优先 (Inter / PingFang / SF)

#### 字号规范
```
Display    20px  SemiBold
Title      16px  Medium
Body       14px  Regular
Caption    12px  Regular
Micro      11px  Regular
```

#### 行高
```
Body  line-height: 1.6
Title line-height: 1.4
```

### 3. 间距系统 (Spacing Scale)
统一 4px 体系：
```
4px   xs
8px   sm
12px  md
16px  lg
24px  xl
32px  xxl
```

### 4. 圆角与阴影
```
Radius-Small  6px
Radius-Medium 8px
Radius-Large  12px
```

阴影（极轻）：
```
Shadow-1: 0 1px 2px rgba(0,0,0,0.04)
Shadow-2: 0 2px 6px rgba(0,0,0,0.06)
```

**不要重阴影，不做"卡片堆叠风"**

## 三、核心组件规范

### 1. Message 组件规范

#### 普通用户消息
- 背景：透明
- 文本颜色：Text-Primary
- 时间戳：Text-Tertiary

#### Agent 消息结构
```
[Header]
  Agent 头像 + 名称 + Badge

[Content]
  结论
  结构化内容（列表/表格/块）

[Meta Bar]
  引用了 X 条上下文 | 创建了 X 个任务 | 写入 X 条记忆
```

**规范**：
- 不使用气泡背景（保持飞书风格）
- Agent 名称颜色：Primary-600
- Meta Bar 字号：12px，Text-Tertiary
- 可点击展开 provenance 抽屉

### 2. Approval Card 规范

- 背景：Background-Panel
- 边框：Border-Default
- 左侧可加状态色条（Danger/Warning）

**结构**：
```
标题：Agent 请求执行操作
├── 摘要说明
├── 影响范围
└── 引用来源

[允许一次] [总是允许] [拒绝]
```

**按钮规则**：
- 主按钮：Primary-500
- 危险动作：Danger-500

### 3. Side Panel 规范 (Memory / Tasks / Agents)
- 宽度：320px – 360px
- 背景：Background-Subtle
- Tab 高度：40px

**卡片样式**：
```
Background-Base
Border-Light
Radius-Medium
内边距 12px
```

### 4. Task 组件规范

**状态色**：
```
Open    → Info-500
Doing   → Primary-500
Done    → Success-500
Blocked → Danger-500
```

**展示**：
```
[ ] 录屏 Demo
    负责人：张三
    截止：2/21
    来源：跳转消息
```

## 四、交互规范

### 1. Hover 显示操作

消息 hover 时显示：
- 回复
- 复制
- 记一笔

### 2. Agent 思考状态

**显示**：`SummaryAgent 正在生成总结…`
**样式**：Text-Tertiary，带微弱 loading 动效
**不要闪烁**

### 3. 可解释性呈现

`"引用了 12 条上下文"`

- 默认折叠
- 点击右侧 Drawer 打开
- Drawer 背景：Background-Base
- 可跳转到原消息

---

## 五、动效原则

- 动效时间：150ms – 200ms
- 只用于：面板展开 / 消息出现 / Approval 出现
- **不做夸张动画**

---

## 六、AI 特有视觉规则

### 1. AI 不喧宾夺主
- 不用大面积色块
- 不做发光/渐变

### 2. AI 输出块状化
- 使用分隔线
- 标题加粗
- 列表化内容

### 3. 可追溯元素统一为灰色注释层

---

## 七、品牌定位视觉总结

**ClawTeam UI 气质应该是**：

> 稳定、可信、结构化、克制、企业级

> 像飞书一样理性，但比飞书更"智能"
